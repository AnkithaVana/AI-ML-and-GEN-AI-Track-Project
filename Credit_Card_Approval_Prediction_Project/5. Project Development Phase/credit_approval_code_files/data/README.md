# Dataset

Place the real Kaggle-style files here when available:

- `application_record.csv`
- `credit_record.csv`

If these files are not present, `train_model.py` automatically creates a generated demo dataset so the project can still train and run locally.

Expected target behavior:

- Applicants with clean credit status are labeled as approved.
- Applicants with overdue credit status values are labeled as rejected/high risk.
